## The Core Idea
Naming a fallacy is step one. Better philosophy repairs the argument structure. If you can only label errors but cannot reconstruct stronger reasoning, critique remains shallow.

This lesson trains argument repair, not just detection. You move from "this is bad reasoning" to "here is a better version that preserves the core concern."

## Why This Matters
Degree-level essays are judged on constructive rigor. Examiners expect you to:
1. identify inferential weakness,
2. preserve what is reasonable in the original concern,
3. rebuild a defensible argument.

## Worked Example
Fallacious claim:
"Either we ban social media or society collapses."

Repair move:
- expand option space (regulation, age-targeted safeguards, platform accountability);
- define evaluation criteria (harm reduction, rights impact, feasibility);
- argue for one option using evidence and constraints.

Repaired conclusion:
"Targeted regulation is preferable to blanket bans because it addresses documented harms while preserving legitimate civic and educational use."

## Common Mistake
Replacing one fallacy with another, for example:
from false dilemma to vague sweeping generalization.

Repairs need explicit structure, not just softer language.

## Before You Write (Step 2)
Use this sequence:
1. Quote or paraphrase a flawed claim.
2. Name the structural issue.
3. Rewrite with clearer options and justified conclusion.
