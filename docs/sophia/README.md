# SOPHIA Documentation

SOPHIA is the showcase and reference application for the Restormel platform.

It exists to prove the platform in a real product: graph-grounded reasoning, verification, retrieval explainability, BYOK and billing flows, API surfaces, and guided philosophy learning all meet here first before they are fully separated into platform modules.

## What SOPHIA is now

- a public-facing showcase app for graph-grounded philosophical reasoning
- a reference implementation for core Restormel platform capabilities
- a dogfooding environment where platform ideas are validated in product form
- the domain-specific experience layer, while Restormel remains the platform/product-family layer

## Current focus

### Active priorities

1. Keep SOPHIA documentation small, current, and aligned to the Restormel platform model.
2. Maintain SOPHIA as a convincing reference application across reasoning, API, and learning surfaces.
3. Continue domain growth with explicit status and promotion criteria rather than parallel undocumented expansions.
4. Extract reusable platform concerns into Restormel over time without letting SOPHIA documentation drift into a second competing platform plan.
5. Preserve historical material for traceability, but keep it out of the active instructional surface.

### Roadmap themes

- Keep SOPHIA credible as the showcase app
- Separate showcase truth from platform truth
- Stabilise domain growth
- Improve evidence and explainability
- Support platform extraction without forking the narrative

### Live showcase domains

- Ethics
- Philosophy of Mind
