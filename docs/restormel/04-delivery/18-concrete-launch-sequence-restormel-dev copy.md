# Restormel Build Pack 05: Concrete Launch Sequence for restormel.dev

## Purpose
Define the staged launch sequence for `restormel.dev`.

## Launch principle
Launch in progressive layers. Each stage should sharpen the same message: Restormel helps teams inspect reasoning, support, contradictions, and run-to-run changes.

## Stage 0: Foundations
### Objective
Lock naming, message hierarchy, and IA.
### Deliverables
- brand and copy system
- sitemap
- homepage draft
- product page outlines

## Stage 1: Public landing site
### Objective
Explain the category and collect interest.
### Deliverables
- homepage
- product pages
- docs stub
- security / about pages

## Stage 2: Playground and samples
### Objective
Let visitors see the workspace value quickly.
### Deliverables
- sample runs
- compare demo
- example lineage export
- docs for imports

## Stage 3: Authenticated workspace
### Objective
Allow real users to import and retain runs.
### Deliverables
- login
- saved runs
- workspace shell
- basic pricing gates

## Stage 4: Team product
### Objective
Support collaboration and monetisation.
### Deliverables
- team sharing
- compare history
- evaluator usage
- billing
- admin basics

## Stage 5: Enterprise and procurement
### Objective
Support larger customers and marketplaces.
### Deliverables
- enterprise controls
- trust materials
- procurement readiness
- marketplace SKU

## Launch metrics
- visitors understand the product in one visit
- demo engagement is strong
- imported runs convert to retained usage
- enterprise conversations are about debugging, evaluation, and lineage, not generic infra
